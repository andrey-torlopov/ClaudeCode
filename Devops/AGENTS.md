# Правила для Codex

## Основное
- Общаемся на русском языке.
- Домен: DevOps, инфраструктура, автоматизация, CI/CD.
- Языки: Bash, Python, YAML, HCL (Terraform), Dockerfile, JSON, TOML.
- Валидация: `shellcheck`, `yamllint`, `terraform validate`, `hadolint`, `ansible-lint`.
- Вся документация и контент скиллов - на русском, если не указано иное.
- Мне нужен строгий наставник, который поможет мне вырасти как инженеру.

## Стиль кода
- Bash: strict mode (`set -euo pipefail`), shellcheck-совместимый, переменные в кавычках.
- YAML: 2 пробела, явные типы (`true`/`false`, не yes/no), без табов.
- Terraform: модули для переиспользования, переменные с описаниями и типами.
- Dockerfile: multi-stage builds, минимальные базовые образы, не root, HEALTHCHECK.
- Python: PEP 8, type hints, виртуальные окружения.
- Все скрипты идемпотентны: повторный запуск не ломает состояние.
- Секреты только через переменные окружения или secret managers, никогда в коде.
- В комментариях не используем длинные тире; не пишем очевидные комментарии.

## Архитектура
- Не предлагаем новые инфраструктурные паттерны без запроса; сохраняем текущую архитектуру.

## RnD
- Для исследовательских результатов спрашиваем путь, куда сохранить Markdown-отчёт.

## Safety и качество
- Удалять ресурсы/файлы только после явного подтверждения.
- Деструктивные операции (`terraform destroy`, `rm -rf`, `DROP DATABASE`) - только с подтверждением.
- Конфиги должны проходить валидацию без правок.
- При расчётах показываем полную формулу с числителем и знаменателем перед результатом.
- Читать можно свободно без подтверждения.
