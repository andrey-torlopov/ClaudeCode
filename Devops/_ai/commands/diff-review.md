Проведи review текущих изменений в git-репозитории в папке $ARGUMENTS и запиши результат в файл .md $ARGUMENTS

Шаги:
1. Выполни `git status -s` чтобы увидеть список изменённых файлов
2. Выполни `git diff` для просмотра unstaged изменений
3. Выполни `git diff --cached` для просмотра staged изменений
4. Прочитай каждый изменённый файл целиком для полного контекста

Проанализируй изменения и дай review по следующим критериям:
- Наличие секретов, паролей, ключей в коде или конфигах
- Безопасность: открытые порты, привилегированные контейнеры, root, chmod 777
- Корректность конфигов (YAML-синтаксис, HCL-валидность, Dockerfile best practices)
- Идемпотентность скриптов (повторный запуск не ломает систему)
- Resource limits для контейнеров и pods
- Health checks (liveness, readiness probes)
- Обработка ошибок в скриптах (set -euo pipefail, trap, exit codes)
- Соответствие конвенциям из CLAUDE.md

Формат ответа:
- Ответ и рассуждения на русском языке
- Краткое описание что изменено
- Список замечаний с указанием файла и строки
- Сначала указываем критичные замечания и блокеры MR, затем по нисходящей
- Кратко укажи что можно улучшить в будущем
- Если всё хорошо - напиши что замечаний нет
