Проведи краткий review текущих изменений в git-репозитории в папке $ARGUMENTS

Шаги:
1. Выполни `git status -s` чтобы увидеть список изменённых файлов
2. Выполни `git diff` для просмотра unstaged изменений
3. Выполни `git diff --cached` для просмотра staged изменений
4. Прочитай каждый изменённый файл целиком для полного контекста

Проанализируй изменения и дай краткое review только по критическим проблемам:
- Секреты, пароли, ключи API в коде или конфигах
- Открытые порты, 0.0.0.0/0 в firewall rules
- Контейнеры без resource limits или от root
- Скрипты без обработки ошибок (нет set -e)
- :latest теги в Docker образах
- chmod 777, --privileged, curl | bash
- Отсутствие health checks

Формат ответа:
- Краткое перечисление что срочно нужно поправить и мержить нельзя
- Список замечаний с указанием файла и строки
- Предложения по улучшению
- Если всё хорошо - напиши что замечаний нет
