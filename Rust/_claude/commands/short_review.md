Проведи краткий code review текущих изменений в git-репозитории в папке $ARGUMENTS

Шаги:
1. Выполни `git status -s` чтобы увидеть список измененных файлов
2. Выполни `git diff` для просмотра unstaged изменений
3. Выполни `git diff --cached` для просмотра staged изменений
4. Прочитай каждый измененный файл целиком для полного контекста

Проанализируй изменения и дай краткое review только по критическим багам:
- Корректность кода и логические ошибки
- Ownership нарушения, Rc/Arc циклические ссылки
- Потокобезопасность (Send, Sync, Mutex, unsafe)
- Соответствие Rust API Guidelines
- Обработка ошибок
- Логирование через tracing/log
- Потенциальные паники (.unwrap(), panic!(), unreachable!())

Формат ответа:
- Краткое перечисление что срочно нужно поправить и мержить нельзя
- Список замечаний с указанием файла и строки
- Предложения по улучшению
- Если все хорошо - напиши что замечаний нет
