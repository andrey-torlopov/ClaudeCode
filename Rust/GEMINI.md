# Правила для Gemini CLI

## Основное
- Общаемся на русском языке.
- Язык проекта: Rust; менеджер пакетов: Cargo.
- Сборка: `cargo build`; тесты: `cargo test`.
- Вся документация и контент скиллов — на русском, если не указано иное.

## Стиль Rust
- Следуем официальным гайдлайнам Rust (Rust API Guidelines).
- Используем идиоматичный код (pattern matching, `Result` и `Option` вместо паник).
- Явное управление ошибками: прокидываем ошибки через `?`.
- Избегаем `unwrap()` и `expect()` в продакшен-коде; обосновываем использование `unsafe`.
- Избегаем излишнего клонирования (`.clone()`), предпочитаем передачу по ссылке, где это возможно.
- Не добавляем избыточные комментарии, если код самодокументируем.

## Архитектура
- Не предлагаем новые архитектурные паттерны без запроса; сохраняем текущую архитектуру проекта.

## RnD
- Для исследовательских результатов спрашиваем путь, куда сохранить Markdown-отчёт.

## Safety и качество
- Деструктивные команды только после явного подтверждения.
- Код должен компилироваться без правок (успешный запуск `cargo check`).
- При расчётах показываем полную формулу с числителем и знаменателем перед результатом.
- Читать можно свободно без подтверждения.

## AI-сетап Gemini
- Базовый промт и оркестратор: `.ai/dev_agent.md`.
- Роли агентов: `.ai/agents/`.
- Скиллы и команды: `.ai/skills/`, `.ai/commands/`; пост-хуки: `.ai/hooks/`.
- Антипаттерны и протоколы: `.ai/rust-antipatterns/`, `.ai/protocols/`.
- Для общих конвенций проекта используем `GEMINI.md`, но исполняем их через сетап в `.ai/`.
