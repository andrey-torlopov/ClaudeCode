# Правила для Codex

## Основное
- Язык проекта: Swift; менеджер пакетов: SPM.
- Сборка: `swift build`; тесты: `swift test`.

## Стиль Swift
- Следуем Swift API Design Guidelines; предпочитаем `let` и value types.
- Используем async/await и структурированную конкуренцию (`TaskGroup`, `async let`); не полагаемся на `Task.detached`.
- UI-код под `@MainActor`; помечаем типы как `Sendable`, когда это возможно.
- Для выходов используем `guard`; избегаем force unwrap и `Any`/`AnyObject` без необходимости.
- В escaping closures используем `[weak self]` и усиливаем `self` только перед первым обращением.
- Не добавляем docstrings, `// MARK:` или `#if DEBUG`, если об этом явно не просят.
- В комментариях не используем длинные тире; не пишем очевидные комментарии.

## Архитектура
- Не предлагаем новые архитектурные паттерны (VIPER, MVC и т.п.) без запроса; сохраняем текущую архитектуру проекта.

## RnD
- Для исследовательских результатов спрашиваем путь, куда сохранить Markdown-отчёт.

## Safety и качество
- Деструктивные команды только после явного подтверждения.
- Код должен компилироваться без правок.
- При расчётах показываем полную формулу с числителем и знаменателем перед результатом.
